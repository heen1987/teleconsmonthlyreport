## 1. 개요: 기존 구조의 문제점

### 이전 구조 (Before)
| 뷰 | 이름 | 문제 |
|---|---|---|
| visual | 통합 대시보드 | 모든 기능이 한 화면에 나열 — 압도적 |
| minutes | 회의록 | 별도 존재하지만 AppView 타입 누락 (TS 오류) |
| review | 검토 | 기능은 있으나 진입 경로가 불명확 |
| admin | 사용자 | 관리자만 접근 가능하지만 섹션 구분 없음 |

### 재설계 목표
1. **파이프라인 시각화** — 현재 내가 어디 있는지 항상 보인다
2. **뷰별 단일 목적** — 각 뷰는 한 가지 일만 한다
3. **온보딩 안내** — 처음 사용자도 흐름을 이해한다
4. **클릭으로 이동** — 파이프라인 단계 클릭 → 관련 뷰로 이동

## 3. 뷰별 콘텐츠 정의

### 3.1 워크스페이스 (`visual`)

**목적:** 홈 화면 — 지금 무엇에 집중해야 하는가?

```
┌──────────────────────────────────────────────────┐
│  PipelineBand (파이프라인 5단계 상태)             │
│                                                  │
│  [🎤 녹음·업로드] → [📋 검토 대기] → [✅ 배포]  │
│       N건                N건             N건      │
│                    | [⚠ 리스크] [✉ 배포실패]      │
├──────────────────────────────────────────────────┤
│  Attention Cards (4개)                           │
│  [프로젝트N] [검토대기N] [오픈리스크N] [배포실패N]│
├─────────────────────────┬────────────────────────┤
│  최근 회의 (6건)        │  프로젝트 (6건)         │
│  - 타이틀 / 상태뱃지    │  - 이름 / 상태          │
│  - 클릭 → 검토 뷰       │  - 컬러 카드            │
├──────────────────────────────────────────────────┤
│  [처음 사용하시나요? ▶] (접힌 온보딩 안내)       │
└──────────────────────────────────────────────────┘
```

**데이터:** `dashboard`, `projects`, `recentMeetings`
**액션:** 파이프라인 단계 클릭 → 해당 뷰 이동, 회의 클릭 → 검토 뷰

---

### 3.2 회의·문서 (`minutes`)

**목적:** 회의 목록 탐색 + 전사 뷰어 + AI 요약

```
┌──────────────────────────────────────────────────┐
│  MinutesConsole (기존 유지)                       │
│  - 노트 목록 (좌측 레일)                          │
│  - 전사 스크립트 (중앙 패널)                      │
│  - AI 요약 + 배포 대상 (우측 패널)                │
└──────────────────────────────────────────────────┘
```

**데이터:** `recentMeetings`, `projects`, `selectedProjectDetail`

---

### 3.3 검토·승인 (`review`)

**목적:** 회의록 검토 → 수정 → 승인 → 배포 (유일한 실제 액션 화면)

┌────────────────┬───────────────────────────────────┐

│  [선택드롭다운]│  [Meeting ID 입력] [불러오기]      │

│  - 최근 회의 처리 상태                            │
│  - Project Knowledge (요약)                       │
│  - Resource Pool + Cost Feedback                  │

**액션:** 비용 후보 승인/반려, 리스크 승격 실행, 이메일 재시도

## 4. PipelineBand 컴포넌트 설계

```
[🎤 녹음·업로드]  →  [📋 검토 대기]  →  [✅ 배포]
     N건                  N건               N건
  (minutes 이동)       (review 이동)    (minutes 이동)

─────── 주의 지표 ───────
[⚠ 오픈 리스크: N건]  [✉ 배포 실패: N건]
  (resources 이동)        (resources 이동)
```

- 모든 단계는 버튼이며 클릭 시 해당 뷰로 이동
- 검토 대기 > 0이면 amber 강조
- 오픈 리스크 > 0이면 red 강조
- 배포 실패 > 0이면 red 강조

## 5. 사용자 흐름 (User Flow)

```
로그인
  │
  ▼
워크스페이스 (홈)
  │  파이프라인 배너로 현황 파악
  │
  ├─ [검토 대기 클릭] ──────────► 검토·승인 뷰
  │                                │
  │                                ├─ Meeting ID 입력
  │                                ├─ 회의록 검토·수정
  │                                ├─ 승인
  │                                └─ 이메일 배포
  │
  ├─ [최근 회의 클릭] ──────────► 검토·승인 뷰 (ID 자동입력)
  │
  ├─ [전체 보기 클릭] ──────────► 회의·문서 뷰
  │                                │
  │                                └─ 회의 목록·전사·AI 요약
  │
  ├─ [오픈 리스크 클릭] ────────► 자원·운영 뷰
  │                                │
  │                                ├─ 리스크 승격 실행
  │                                └─ 비용 후보 승인/반려
  │
  └─ [지식베이스] ──────────────► 지식베이스 뷰
                                   │
                                   └─ 프로젝트별 지식 검색
```
