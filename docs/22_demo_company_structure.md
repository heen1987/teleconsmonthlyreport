# 50명 SW 개발회사 데모 조직 구조

이 문서는 AI-PMS 시연/개발 DB에 넣을 기준 회사 모델을 정의한다.
회의 녹음 앱은 참석자를 직접 선택하지 않고 프로젝트만 선택하므로, 각 프로젝트의
`project_members`가 회의록 자동 배포 대상의 원천 데이터가 된다.

## 회사 기준

| 항목 | 값 |
| --- | --- |
| 회사명 | 새싹SW |
| 업종 | SW 개발 |
| 연매출 | 50억 원 |
| 총 인원 | 50명 |
| 개발 인원 | 45명 |
| 프로젝트 | 15개 |
| 본부 | 4개 |
| 데모 총 연봉 | 약 35.12억 원 |
| 개발직 총 연봉 | 약 31.11억 원 |

| 본부 | 성격 | 인원 | 구성 |

| 플랫폼개발본부 | 개발 수행조직 | 15명 | 본부장/실장/팀장/개발자 |
| 서비스개발본부 | 개발 수행조직 | 15명 | 본부장/실장/팀장/개발자 |

## 직위와 직책

| 구분 | 체계 |
| --- | --- |
| 직위 | 사원, 선임, 책임, 수석, 이사 |
| 팀장 | 책임급 |
| 실장 | 수석급 |
| 본부장 | 이사급 |

AI연구소의 최상위 직책은 명칭만 `연구소장`으로 두고, 직위와 권한 레벨은
본부장과 동일하게 이사급으로 취급한다.

## 개발 수행조직 1개 본부당 편제

| 직위 | 직책 | 인원 |
| --- | --- | ---: |
| 이사 | 본부장 또는 연구소장 | 1명 |
| 수석 | 실장 | 2명 |
| 책임 | 팀장 | 3명 |
| 선임 | 개발자 | 4명 |
| 사원 | 개발자 | 5명 |

각 개발 수행조직은 15명이고, 5개 프로젝트를 담당한다. 프로젝트 1개당
기본 배정 인원은 3명이며, 자동 배포 대상은 이 3명의 프로젝트 구성원이다.

## 연봉과 투입 기준

데모 데이터는 실제 개인정보가 아닌 임의 구성원명과 직위 기반 연봉을 사용한다.
연봉은 `resource_profiles.metadata.annual_salary_krw`에 저장하고, 프로젝트
참여 시점의 연봉 스냅샷과 배정 원가는 `project_members`에 저장한다.

| 직위 | 기준 연봉 |
| --- | ---: |
| 사원 | 4,200만 원 |
| 선임 | 5,500만 원 |
| 책임 | 7,000만 원 |
| 수석 | 8,800만 원 |
| 이사 | 1억 1,800만 원 |

직책 수당은 대표이사, 본부장/연구소장, 실장, 팀장, 재무담당에 데모 기준으로
가산한다. 개발직은 개발직 보정수당을 추가한다.

| 프로젝트 역할 | 투입률 | 계획 M/M |
| --- | ---: | ---: |
| project_lead | 60% | 0.6 |
| technical_lead | 80% | 0.8 |
| developer | 100% | 1.0 |

프로젝트 1개당 기본 투입 합계는 240%, 2.4 M/M이다.

## 생성 스크립트

계획 JSON만 생성:

```bash
backend/.venv/bin/python scripts/seed_demo_company.py
```

DB에 실제 반영:

```bash
backend/.venv/bin/python scripts/seed_demo_company.py --apply
```

생성 결과는 `runtime/demo_company/latest_plan.json`에 저장된다.
DB 반영 시 사용자, 인력 리소스, 프로젝트, 프로젝트 구성원, 연봉 스냅샷,
투입률, 계획 M/M, 배정 원가를 upsert한다.
계정 기본 비밀번호는 `1234`, 기본 상태는 `active`다.

## 자동 배포 기준

회의 앱에서 프로젝트를 선택하면 다음 데이터 흐름을 따른다.

```text
프로젝트 선택
  -> project_members 조회
  -> 프로젝트 구성원 자동 배포 대상 확인
  -> 녹음/업로드
  -> 분석 결과 생성
  -> project_members의 활성 사용자 이메일로 회의록 자동 배포
```
