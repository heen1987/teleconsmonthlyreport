휴대폰과 태블릿에서 같은 APK를 설치한 뒤 녹음, 업로드, 처리상태 확인 흐름을 점검합니다.

설치 전 확인

  1. 테스트 기기에서 외부 APK 설치를 허용합니다.
  2. 기존 com.aipms debug 앱이 있으면 필요 시 삭제합니다.
  3. 네트워크가 현재 public Platform/Collection API에 접근 가능한지 확인합니다.
  4. 설치 후 앱 이름이 AI-PMS Recorder로 보이는지 확인합니다.

휴대폰 확인

  • 세로 화면에서 프로젝트 선택, 자동 배포 대상 확인, 녹음 컨트롤이 한 컬럼 흐름으로 보이는지 확인합니다.
  • 긴 프로젝트명과 상태 문구가 버튼이나 카드 밖으로 넘치지 않는지 확인합니다.
  • 녹음 시작, 중지, 업로드 버튼이 엄지 조작 범위에서 접근 가능한지 확인합니다.

태블릿 확인

  • 600dp 이상 화면에서 프로젝트 선택/자동 배포 대상 영역과 녹음/상태 영역이 두 컬럼으로 나뉘는지 확인합니다.
  • 가로/세로 전환 후에도 버튼, 상태, 목록이 겹치지 않는지 확인합니다.
  • 회의 상태 확인 영역이 업로드 완료 후에도 안정적으로 유지되는지 확인합니다.

기능 흐름 확인

  1. 사번으로 로그인하고 필요 시 초기 비밀번호를 변경합니다.
  2. 프로젝트를 선택하고 프로젝트 구성원이 자동 배포 대상임을 확인합니다.
  3. 짧은 테스트 음성을 녹음한 뒤 업로드합니다.
  4. Collection job이 생성되고 처리상태가 조회되는지 확인합니다.
  5. Web 콘솔에서 해당 회의가 검토 대상으로 이어지는지 확인합니다.
