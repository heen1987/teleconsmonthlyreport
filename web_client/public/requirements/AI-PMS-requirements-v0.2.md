## 1. 개요

### 1.1. 문서 목적

본 문서는 로컬 LLM 기반 AI-PMS 플랫폼 개발을 위한 시스템 범위, 사용자 요구, 기능 요구사항, 데이터 요구사항, 외부 인터페이스 요구사항, 비기능 요구사항, 수용기준을 정의하기 위한 개발용 요구사항 정의서임.

본 문서는 개발팀이 기능 구현, 화면 설계, API 설계, 데이터 모델링, 테스트 시나리오 작성 시 기준 문서로 활용함. 특히 이번 과제의 구현/검증 범위와 후속 확장 범위를 구분하는 기준 문서로 사용함.

### 1.2. 시스템 목적

본 시스템은 회의 음성 및 관련 자료를 수집하고, 로컬 LLM 기반 AI 분석을 통해 회의록 초안과 후속 업무 후보를 생성한 뒤, 사용자의 검토·승인을 거쳐 PMS 실행정보로 반영하는 것을 목적으로 함.

시스템의 핵심은 회의록 자동 작성 자체가 아니라 회의내용을 업무, 결정사항, 리스크, 자원 요청, 지식관리 데이터로 전환하는 것임.

### 1.3. 시스템 범위

| 구분 | 내용 |
|---|---|
| 포함 범위 | 사번 기반 로그인, 프로젝트 선택, 회의 음성 업로드, 파일 검증, 분석작업 생성, AI 분석, 회의록 검토·승인, 이메일 배포, PMS 실행정보 반영, 감사로그 |
| 제외 범위 | 참석자 선택 필수화, 화자 매핑, 발언자/담당자 임의 추정, 실시간 자막, AI 분석결과 자동 확정, 공개 회원가입, ERP·HCM 실시간 연동, 문서관리 고도화, 알림 고도화, 칸반보드 전체 기능 |
| 개발 기준 | MVP 중심 구현, 후속 확장 가능 구조 유지 |
| 주요 연결 기준 | Project_ID, Meeting_ID, source_id |

## 2. 플랫폼 개요

### 2.1. 플랫폼 개발 목적

본 플랫폼은 회의 음성 및 관련 자료를 수집하고, AI 분석을 통해 회의록 초안과 후속 업무 후보를 생성한 뒤, 사용자 검토·승인을 거쳐 PMS 실행정보로 반영하기 위한 업무지원 플랫폼임.

개발 목적은 단순 회의록 자동 작성이 아니라, 회의내용을 프로젝트 운영정보로 활용할 수 있도록 업무·결정·리스크·자원·지식 데이터로 전환하는 것임.

### 2.2. 플랫폼 적용 범위

| 구분 | 내용 |
|---|---|
| 포함 범위 | 로그인, 프로젝트 선택, 회의정보 업로드, 파일 검증, AI 분석, 회의록 검토·승인, 이메일 배포, PMS 실행정보 반영, 감사로그 기록 |
| 제외 범위 | 참석자 선택 필수화, 화자 매핑, 발언자/담당자 임의 추정, 실시간 자막, AI 자동 확정, ERP 실시간 연동, 문서관리 고도화, 칸반보드 전체 기능 |
| 주요 연결 기준 | Project_ID, Meeting_ID, source_id |
| 개발 기준 | MVP 중심 구현, 후속 확장 가능 구조 유지 |

### 2.3. 사용자 유형 및 역할

| 사용자 유형 | 설명 | 주요 기능 |
|---|---|---|

| 프로젝트 관리자 | 회의록을 검토·승인하고 PMS 반영을 수행하는 사용자 | 회의록 검토·수정·승인, 이메일 배포, Action Item·Risk 반영 |
| 시스템 관리자 | 사용자와 권한을 관리하는 사용자 | 사용자 등록, 계정 상태관리, 프로젝트 권한 관리 |
| 운영 담당자 | 분석작업과 오류 상태를 확인하는 사용자 | 분석 상태조회, 실패 원인 확인, Retry 확인, 감사로그 조회 |

## 9. MVP 범위 통제 기준(Scope Gate)

새 기능을 MVP에 포함하기 전 다음 기준을 모두 확인함.

| 번호 | 판단 질문 |
|---|---|
| 1 | 회의 업로드부터 승인·배포·PMS 반영까지의 핵심 흐름을 직접 강화하는가? |
| 2 | Project_ID 또는 Meeting_ID를 기준으로 추적 가능한가? |
| 3 | AI 결과가 자동 확정되지 않고 승인 단계를 통과하는가? |
| 4 | 참석자 선택 필수화, 화자 매핑, 공개 회원가입, ERP 실시간 연동을 다시 끌어오지 않는가? |
| 5 | 이번 과제에서 검증 가능한 API, 화면, 테이블, smoke test가 있는가? |

위 질문 중 하나라도 명확하지 않으면 후속 확장 백로그로 분리함.
